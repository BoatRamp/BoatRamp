# Packaging assets

Shared, install-surface-independent payload for the OS-native packages (DEB / RPM
/ Nix / Homebrew) and for hand installs. Packaging is *delivery, not divergence*:
every package wraps the **same** `boatramp` binary and points at the **same**
config and docs —
these files only change how it lands on disk and gets started.

## Layout

| Path | What | Consumed by |
|---|---|---|
| `systemd/boatramp.service` | Hardened service unit (static/handler/TLS/cluster profile; compute needs a documented drop-in) | DEB/RPM/Nix, hand install |
| `systemd/boatramp.sysusers` | `sysusers.d` snippet — the `boatramp` system user | DEB/RPM |
| `systemd/boatramp.tmpfiles` | `tmpfiles.d` snippet — `/etc/boatramp` + `/var/lib/boatramp` | DEB/RPM |

Shell completions and the man page are **generated by the binary itself** so they
can't drift from the real flags — packages run them at build time:

```sh
boatramp completions bash > boatramp.bash
boatramp completions zsh  > _boatramp
boatramp completions fish > boatramp.fish
boatramp man              > boatramp.1
```

## Hand install (Linux, systemd)

```sh
install -Dm755 boatramp                 /usr/bin/boatramp
install -Dm644 systemd/boatramp.service /etc/systemd/system/boatramp.service
install -Dm644 systemd/boatramp.sysusers /usr/lib/sysusers.d/boatramp.conf
install -Dm644 systemd/boatramp.tmpfiles /usr/lib/tmpfiles.d/boatramp.conf
install -Dm644 boatramp.cfg.example     /etc/boatramp/boatramp.cfg
systemd-sysusers && systemd-tmpfiles --create
systemctl daemon-reload && systemctl enable --now boatramp
```

The unit's default sandbox is tuned for static hosting + wasm handlers + TLS +
the cluster client. Running the **compute** backends (microVM/container: `/dev/kvm`,
namespaces, mounts) needs a relaxed drop-in — see the commented block at the end of
`boatramp.service`.

## NixOS (flake module)

The flake exposes an overlay + a `services.boatramp` module (`nix/package.nix`,
`nix/nixos-module.nix`) that renders the same hardened unit declaratively:

```nix
{
  inputs.boatramp.url = "github:BoatRamp/BoatRamp";
  # …in a host config:
  imports = [ inputs.boatramp.nixosModules.default ];
  nixpkgs.overlays = [ inputs.boatramp.overlays.default ];
  services.boatramp = {
    enable = true;
    configFile = pkgs.writeText "boatramp.cfg" ''(serve: (addr: "0.0.0.0:8080"))'';
    openFirewall = true;      # opens 80 + 443
    # compute = true;         # relax the sandbox for the microVM/container backends
  };
}
```

A `nixosTest` (`nix flake check` on Linux) boots a VM with the service enabled and
asserts the hardened unit starts + binds.

## Package metadata

Version, license, description, and asset names are single-sourced from
`[workspace.package]` in the top-level `Cargo.toml`; the `cargo-deb` /
`cargo-generate-rpm` metadata lives in `crates/boatramp/Cargo.toml`
(`[package.metadata.*]`) and reads from there. Packages never restate them.
